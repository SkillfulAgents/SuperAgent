# Notifications Steps

Notifications are generated by agent activity (e.g. agent responses, requests).
The badge and "Mark all read" button only appear when there are unread notifications.

## verify-notification-badge

Look at the notification bell icon in the sidebar footer.
Take a snapshot.
If there are unread notifications, a red badge with a number should appear on the bell icon.
If no badge is visible, it means all notifications are already read — this is normal, not a bug.
Assert: the bell icon is present in the sidebar footer.

---

## open-notifications

Click the notification bell icon in the sidebar footer.
Take a screenshot.
Assert: the notifications popover opens. It shows either a list of notifications or "No notifications yet".

---

## mark-all-read

In the notifications panel, look for a "Mark all read" button next to the "Notifications" heading.
Note: this button only appears when there are unread notifications (unreadCount > 0). If all notifications are already read, the button will not be visible — this is expected behavior, not a bug.
If the button is visible, click it and verify the badge disappears.
If the button is not visible, note that all notifications are already read and move on.

---

## click-notification

In the notifications panel, if there are any notification items, click the first one.
Take a screenshot.
Assert: the main content navigates to the session referenced by the notification, and the notification is marked as read (blue dot disappears).
If no notifications exist, skip this step.
