# Notifications

Notifications are generated by agent activity such as agent responses and interaction requests. They are accessed from a bell icon in the sidebar footer and displayed in a popover panel.

## Notification Bell

The bell icon serves as the entry point and provides an at-a-glance unread count.

### Components

- **Bell icon**: located in the sidebar footer. Always visible.
- **Unread badge**: a red badge overlaying the bell icon, displaying the number of unread notifications. Only visible when `unreadCount > 0`.

### Interactions

- Clicking the bell icon opens the notifications popover.

## Notifications Popover

A panel that lists all notifications for the current user.

### Components

- **Notification list**: a scrollable list of notification items, or a "No notifications yet" empty state if none exist.
- **Notification item**: each item displays a summary of the notification. Unread items are marked with a blue dot indicator.
- **"Mark all read" button**: appears next to the "Notifications" heading. Only visible when there are unread notifications (`unreadCount > 0`).

### Interactions

- Clicking a notification item navigates the main content area to the session referenced by that notification and marks the notification as read (the blue dot disappears).
- Clicking "Mark all read" marks all notifications as read, removing the unread badge from the bell icon. The button disappears once all notifications are read.
